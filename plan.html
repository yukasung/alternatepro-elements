1. ภาพรวมโมดูล Header Footer Builder

โมดูลนี้เพิ่มความสามารถให้ผู้ดูแลเว็บสร้าง Header และ Footer ด้วย Elementor Free ผ่าน template post type ของปลั๊กอินเอง

สร้าง Template

ใช้ CPT apro_template สำหรับเก็บ Header/Footer templates และเปิดให้แก้ไขด้วย Elementor

เลือก Template อัตโนมัติ

ระบบเลือก template จาก page override, display conditions, ภาษา และ priority

ทำงานกับ Theme Fallback

ถ้าไม่มี custom template ที่ match โมดูลจะไม่ render อะไร เพื่อให้ fallback ของธีมทำงานตามเดิม

2. เป้าหมายของระบบ

เป้าหมายหลักคือให้ทีมสร้างและควบคุม Header/Footer ได้จาก WordPress Admin โดยยังคงความเบา ปลอดภัย และยืดหยุ่น

Functional Goals

  • สร้าง Header และ Footer templates ได้
  • แก้ไข template ด้วย Elementor Free
  • รองรับ display conditions และ priority
  • รองรับ override ระดับ page/post
  • รองรับเว็บหลายภาษาแบบไม่บังคับติดตั้ง plugin เพิ่ม

Engineering Goals

  • แยกโมดูลใน includes/modules/header-footer/
  • ใช้ metabox native ของ WordPress ก่อน
  • โหลด assets เฉพาะหน้าที่เกี่ยวข้อง
  • ไม่ fatal เมื่อ Elementor ไม่ active
  • เก็บ matched template cache ต่อ request

3. ความสัมพันธ์กับ AlternatePro Theme

โมดูล render เฉพาะ hook หลักของธีม และไม่แตะ output fallback หากไม่มี template ที่ match

Theme Hook สถานะใช้งาน บทบาท
alternatepro_before_header ไม่ render สงวนไว้สำหรับธีมหรือ extension อื่น
alternatepro_header render header แสดง custom Header template เมื่อมี match
alternatepro_after_header ไม่ render สงวนไว้สำหรับธีมหรือ extension อื่น
alternatepro_footer render footer แสดง custom Footer template เมื่อมี match
alternatepro_before_content, alternatepro_after_content ไม่ render อยู่นอก scope ของ Header/Footer module
หมายเหตุ: Page override ค่า none จะไม่ render custom template และมี filter ให้ธีมใช้ตรวจ suppress fallback ได้ แต่การซ่อน fallback จริงขึ้นกับการเชื่อมต่อในธีมด้วย

4. Elementor Requirements

ใช้ requirements ของ Elementor เป็น compatibility baseline และต้อง fail gracefully หาก Elementor ไม่พร้อมใช้งาน

รายการ ขั้นต่ำ แนะนำ พฤติกรรม
WordPress 6.5+ ล่าสุด แสดง admin notice หากต่ำกว่าขั้นต่ำ
PHP 7.4+ 8.2+ ไม่โหลด module หาก core requirements ไม่ผ่าน
Database MySQL 5.6+ หรือ MariaDB 10.5+ ตาม hosting ที่รองรับ WordPress ล่าสุด ไม่มี query เฉพาะ database vendor
WP Memory Limit 256MB 512MB เลี่ยง query ซ้ำและไม่โหลด frontend assets ทุกหน้า
Elementor Elementor Free เวอร์ชันล่าสุด ไม่ fatal หาก inactive และ render เฉพาะเมื่อ frontend API พร้อม

7. Custom Post Type: apro_template

CPT นี้เป็นที่เก็บ Elementor templates สำหรับ Header และ Footer โดยเปิด UI ใน admin แต่ไม่เปิดเป็น public content

Setting Value เหตุผล
Public false Template ไม่ใช่หน้า public โดยตรง
Show UI/Menu true ให้ admin สร้างและจัดการ template ได้
Show in REST true รองรับ block/editor ecosystem และ Elementor compatibility
Supports title, editor, elementor, revisions แก้ไขด้วย Elementor และย้อน revision ได้
Menu Icon dashicons-layout สื่อความหมายว่าเป็น layout/template

8. Template Meta Fields

Meta fields เป็นข้อมูลควบคุมการเลือก template และต้อง sanitize ทุกครั้งก่อนบันทึก

Meta Key Type Values การใช้งาน
_apro_template_type string header, footer ระบุว่า template render ที่ hook ใด
_apro_template_status string active, inactive template ต้อง active จึงจะ render
_apro_template_priority integer 0 ขึ้นไป ใช้ตัดสินเมื่อ condition specificity เท่ากัน
_apro_display_conditions JSON string include/exclude rules resolver แปลงเป็น array ต่อ request
_apro_language string all, th, en, detected language เลือก template ตามภาษาปัจจุบัน แล้ว fallback เป็น all

9. Display Conditions

Conditions ใช้เลือก template ตามบริบทของ request โดย exclude rules มี priority สูงกว่า include rules

Include Rules

  • Entire Site
  • Front Page
  • All Pages และ Specific Pages
  • All Posts และ Specific Posts
  • Archives, Search Results, 404 Page
  • URL Path Starts With และ URL Path Contains

Exclude Rules

  • Exclude Specific Pages
  • Exclude Specific Posts
  • ถ้า exclude match จะไม่ render template นั้นทันที
  • เหมาะสำหรับยกเว้น landing page หรือ post เฉพาะรายการ

10. Page-Level Override

Posts และ Pages มี metabox ชื่อ AlternatePro Layout เพื่อกำหนด Header/Footer เฉพาะหน้า

Field Values Behavior
Header Template default, none, template ID Default ใช้ resolver, None ไม่ render custom header, ID render template นั้น
Footer Template default, none, template ID Default ใช้ resolver, None ไม่ render custom footer, ID render template นั้น

11. Multilingual Support

ระบบรองรับ multilingual แบบ optional integration โดยไม่บังคับติดตั้ง plugin ภาษา

  1. Polylangตรวจ pll_current_language() และ language list จาก Polylang หากมี
  2. WPMLใช้ filter wpml_current_language และ wpml_active_languages เมื่อพร้อมใช้งาน
  3. TranslatePressอ่าน current language จาก function/global/settings เมื่อมี
  4. URL Prefixfallback จาก prefix เช่น /th/, /en/, /jp/, /zh/
  5. All Languagesถ้าไม่พบภาษา ใช้ all เป็นค่า fallback

12. Rendering Flow

Renderer ทำงานผ่าน hook ของธีมและเรียก Elementor frontend API เฉพาะเมื่อมี template ที่ใช้งานได้

Priority Flow

  • Page Override
  • Display Conditions
  • Default Site Template
  • Theme Fallback

Elementor Rendering

  • ตรวจ \Elementor\Plugin::$instance->frontend ก่อน render
  • ใช้ get_builder_content_for_display( $template_id, true )
  • ไม่ double escape Elementor HTML output
  • ป้องกัน duplicate rendering ต่อ type ใน request เดียวกัน
  • enqueue frontend assets เฉพาะเมื่อ render สำเร็จ

13. Priority Logic

การเลือก template ต้องคาดเดาได้และไม่ทำ query ซ้ำโดยไม่จำเป็น

ลำดับ กฎ ผลลัพธ์
1 Page override เป็น template ID render template ID นั้น ถ้า active และ type ถูกต้อง
2 Page override เป็น none ไม่ render custom template และแจ้ง hook/filter สำหรับ theme fallback suppression
3 Language-specific template match เลือกภาษาปัจจุบันก่อน all
4 Exclude condition match ตัด template นั้นออกทันที
5 Specific condition match specific pages/posts ชนะ broad rules
6 Priority number สูงกว่า ชนะเมื่อ specificity เท่ากัน
7 ไม่มี match ไม่ render และปล่อย theme fallback

14. Security Checklist

ทุก input จาก admin ต้อง sanitize และทุก output ใน admin ต้อง escape

Input

  • ใช้ wp_nonce_field() ใน metabox
  • ตรวจ check_admin_referer() ตอน save
  • ตรวจ current_user_can( 'edit_post' )
  • sanitize ด้วย sanitize_text_field(), sanitize_key(), absint()

Output

  • escape admin output ด้วย esc_html(), esc_attr(), esc_textarea()
  • ไม่ใช้ eval()
  • ไม่เปิด arbitrary PHP execution
  • Elementor HTML output ไม่ double escape แต่ต้อง render เฉพาะ template ID ที่ตรวจแล้ว

15. Performance Checklist

โมดูลต้องเบาและไม่เพิ่มงานให้ทุก frontend request โดยไม่จำเป็น

  • cache matched template ID ต่อ request
  • query เฉพาะ active templates ตาม type
  • ใช้ fields => ids และ no_found_rows => true
  • โหลด Elementor frontend rendering เฉพาะเมื่อมี template ID
  • enqueue frontend CSS/JS เฉพาะเมื่อมีการ render template
  • ป้องกัน duplicate render ต่อ header/footer ใน request เดียว
