# Done Workflow

Use this document after completing any task.

Command:

Read `docs/done.md`

## Step 1 - Review

Run:

Read [docs/agents/review.md](agents/review.md)

Verify:

- Architecture compliance
- Scope compliance
- Coding standards
- Documentation requirements

Review must pass before continuing.

## Step 2 - Testing

Run:

Read [docs/agents/test.md](agents/test.md)

Verify:

- Functional Testing
- Unit Testing, if applicable
- Integration Testing, if applicable
- Regression Testing

Testing must pass before continuing.

## Step 3 - Security Review

Run:

Read [docs/agents/security.md](agents/security.md)

Verify:

- WordPress Security
- Elementor Security
- Admin Security
- Data Validation
- Output Escaping

Security review must pass before continuing.

## Step 4 - Documentation Updates

Update:

- [docs/status.md](status.md)
- [CHANGELOG.md](../CHANGELOG.md)

Update additional documents if affected.

Examples:

- `docs/releases/*`
- `docs/testing/*`
- `docs/reviews/*`

## Step 5 - Verify Project Status

Verify:

- Current Phase
- Completed Work
- Remaining Work
- Open Issues
- Next Task

Update [docs/status.md](status.md) if required.

## Step 6 - Completion Summary

Provide:

### Files Created

### Files Modified

### Documentation Updated

### Issues Found

### Outstanding Work

### Recommended Next Task

## Definition of Done

A task is considered complete only when:

✓ Review completed

✓ Testing completed

✓ Security review completed

✓ Documentation updated

✓ Status updated

✓ Changelog updated

✓ Completion summary generated

## Final Output

State one of:

READY FOR NEXT TASK

or

REQUIRES FIXES

Do not start the next task automatically.

End of workflow.
